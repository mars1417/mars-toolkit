🔧 技术说明

内存盘缓存通过创建 RAM Disk（内存盘）并修改 Chrome/Safari 的浏览器缓存路径来实现。使用 macOS 内置的 hdiutil 命令创建 128MB-256MB 内存盘，通过 shell 脚本和 launchd plist 实现开机自动挂载和缓存目录软链接。Chrome 通过 --disk-cache-dir 参数配置缓存路径，Safari 通过修改 plist 中的 WebKit 缓存目录。所有日志保存在 ~/.mars-toolkit/logs/ram-disk-cache.log。

❓ 常见问题

内存盘占了多少内存？会不会影响日常使用？
默认分配128MB，最多256MB。对于4GB以上的Mac完全不影响日常使用。而且内存盘只在有缓存数据时才会占用空间，平时几乎是空的。如果你的内存比较紧张（比如4GB机型），可以手动调整为64MB。
重启后缓存会消失吗？浏览器不白缓存了吗？
会消失，但这是好事。浏览器缓存本身就是临时的——存的是图片、CSS、JS等静态资源。重启后重新加载一次，之后又恢复加速效果。相比SSD缓存带来的频繁写入损耗，重启后一次重新加载的成本可以忽略不计。
支持哪些浏览器？
默认支持 Chrome 和 Safari。Chrome 通过启动参数 --disk-cache-dir 重定向缓存路径；Safari 通过修改 WebKit 偏好设置。Firefox 用户可以手动配置 about:config 中的 browser.cache.disk.parent_directory 参数。Edge/Chromium 内核浏览器同样支持。
