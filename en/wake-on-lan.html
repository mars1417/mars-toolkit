🔧 Technical Details

Wake-on-LAN uses macOS built-in WOL (womp) support, enabled via pmset. A launchd daemon runs in the background, listening for both WOL Magic Packets and HTTP wake requests. Network configuration is recorded for reference when sending wake commands from external devices. All logs saved to ~/.mars-toolkit/logs/wake-on-lan.log.

❓ FAQ

Can it wake from full shutdown?
Yes. When the Mac is plugged in and connected via Ethernet or Wi-Fi, WOL can wake it from sleep, deep sleep, or even full shutdown (as long as the power adapter is connected and the network interface supports it). The script automatically enables this.
Can I wake from outside my home network?
You'll need a tunnel or VPN to your home network first. It works perfectly within the local network. For external access, pair with Ocean VPN or a cpolar tunnel + Tunnel Guardian to reach your LAN, then send the WOL packet.
What devices can send wake commands?
Any device that can send UDP broadcast packets or HTTP requests: phones (iOS/Android via apps or SSH), another computer, your router, smart home hubs, etc. For phones, use SSH + an app like Termux, or simply visit the HTTP port in a browser.
