🔧 Technical Details

Disk Health is a macOS background monitoring daemon. It reads S.M.A.R.T. status from all physical disks (via diskutil info), checks TRIM status (via system_profiler SPNVMeDataType), and monitors root partition disk usage. Checks run every 60 minutes by default, with warnings logged when usage exceeds 85%. Supports manual runs for full reports and large file scanning.

❓ FAQ

My Mac has an HDD, not SSD. Does this work?
Yes. HDDs also have S.M.A.R.T. monitoring — only TRIM is SSD-specific. The tool auto-detects your disk type and shows relevant information.
What is S.M.A.R.T.?
S.M.A.R.T. (Self-Monitoring, Analysis and Reporting Technology) is a built-in health detection system in hard drives. If S.M.A.R.T. shows "Failing", your drive could die at any moment — time to back up NOW. This tool saves you from having to check manually.
What happens if TRIM is disabled?
TRIM is the SSD's garbage collection mechanism. Without it, deleted blocks aren't marked as reusable — write performance degrades over time and SSD lifespan shortens. This tool checks and reminds you to enable it.
Does it consume many resources?
Barely any. The daemon doesn't run continuously — launchd triggers a check every 60 minutes. Each check takes under 1 second. Virtually zero CPU and memory usage.
