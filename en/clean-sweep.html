🔧 Technical Details

Clean Sweep is a macOS background daemon powered by launchd with both timer and event-driven modes. By default it runs daily at 2AM: closes memory-heavy apps (WPS, Stocks, etc.), closes idle Chrome windows, then calls sudo purge to release inactive pages. Browser cache cleaning supports Chrome and Safari. All operations logged to ~/.mars-toolkit/logs/.

❓ FAQ

Will this delete my browser data?
No. Only Cache and Code Cache directories are cleaned. Bookmarks, passwords, extensions, and browsing history are never touched. Same for Safari — only the cache directory.
Is sudo purge safe?
Completely safe. purge is a built-in macOS command that force-releases inactive memory pages — pages the system would reclaim anyway when needed. It just does it immediately. No crashes or data loss.
Does it work on 4GB Macs?
Very noticeably. Tested on a 2017 MacBook Air 4GB — each cleanup cycle frees ~600-800MB of available memory. Safari with 10 tabs no longer stutters, app switching becomes snappy again.
Can I customize what gets cleaned?
Yes. The config file at ~/.mars-toolkit/clean-sweep.conf lets you set the schedule, memory threshold, which browser caches to clean, and whether to clean system logs and trash.
